# Ecosystem

This folder is the map room for Church of AI.

Use it to understand what the independent projects are, what order to read them
in, and when a cross-repo bridge is actually warranted.

Church of AI maps and routes. It does not govern other repos, ratify findings,
or import another repo's claims.

## Start Here

- [MAP.md](MAP.md) - the public ecosystem map.
- [SEQUENCE.md](SEQUENCE.md) - reading paths for different kinds of readers.
- [REPOS.md](REPOS.md) - direct descriptions of the independent projects.
- [BRIDGE-GUIDE.md](BRIDGE-GUIDE.md) - when cross-repo connections should and
  should not be made.
- [ARCHITECTURE.md](ARCHITECTURE.md) - the practical local / bridge / global
  orientation model.
- [ECOSYSTEM-ORIENTATION-AND-REPOSITORY-ARCHITECTURE-REPORT.md](ECOSYSTEM-ORIENTATION-AND-REPOSITORY-ARCHITECTURE-REPORT.md)
  - the longer proposal and reasoning artifact behind the architecture note.

## Boundaries

Use this folder for orientation, not authority transfer.

- Technical claims belong in the owning research repo.
- Governance legitimacy questions belong with the repo or project studying
  legitimacy.
- Participation readiness is controlled by [STATUS.md](../STATUS.md),
  [ROADMAP.md](../ROADMAP.md), and [participation/](../participation/), not by
  ecosystem-map language.
- Cross-repo relationships should stay small, explicit, evidenced, and
  reversible.

If a bridge would make the ecosystem sound more centralized, more proven, or
more governed than it is, do not build it without explicit review.
