# Participation

This folder keeps the participation boundary clear.

Start with [NOT-YET.md](NOT-YET.md). That is the current answer.

[FUTURE-PATHWAYS.md](FUTURE-PATHWAYS.md) sketches what participation could
look like later. It is not an invitation, launch plan, governance process,
membership process, reward system, or contribution guide.

The honest answer right now is that participation pathways are deferred.

This page will change when that changes. Until then, the most useful thing you
can do is read the ecosystem, follow the research repos, and come back when
there is something to participate in.

---

## Start here

- [Not yet](NOT-YET.md) - what is not open, why, and what would need to be true
  before participation pathways open.
- [Future pathways](FUTURE-PATHWAYS.md) - possible future routes, explicitly
  subordinate to the current "not yet" boundary.
- [Roadmap](../ROADMAP.md) and [status](../STATUS.md) - the current public
  state and prerequisites.
- [Claim status](../boundaries/CLAIM-STATUS.md) - how to read claims that sound
  more ready than the evidence allows.

## What participation will eventually include

- Contributing to open-source tools
- Participating in governance discussions
- Proposing and testing rules
- Contributing to research workflow documentation

None of these are open yet. See [NOT-YET.md](NOT-YET.md) for why, and [FUTURE-PATHWAYS.md](FUTURE-PATHWAYS.md) for what would need to be true before they are.

## Reading rule

Read the current answer before the future sketch. If a phrase in this folder
sounds like an invitation to participate now, check [NOT-YET.md](NOT-YET.md)
and [STATUS.md](../STATUS.md) first.
